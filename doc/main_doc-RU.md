## 🏗️ Архитектура системы

Проект построен на **серверной архитектуре с выделенным фоновым ядром**. Админ-панель не является внешним клиентом — она представляет собой локальный интерфейс управления хостом, доступный только владельцу сервера.

1. **Фоновое ядро (Блокчейн-нода):** Работает в фоне как демон (`server.py`). Управляет состоянием цепи, занимается майнингом и предоставляет внутреннее API на базе **FastAPI**.
2. **Панель администратора (Консоль хоста):** Интегрированный текстовый интерфейс (`main.py`) для прямого управления нодой на текущем сервере. Общается с фоновым ядром локально через `127.0.0.1`.

---

## 🛠️ Описание API Эндпоинтов (FastAPI)

Локальный сервер ноды принимает запросы по адресу `http://127.0.0.1:8000`.

### 1. Статус ноды
* **Метод:** `GET`
* **Эндпоинт:** `/status`
* **Возвращает:** Текущую сложность и количество блоков.

### 2. Получение цепи
* **Метод:** `GET`
* **Эндпоинт:** `/blocks`
* **Возвращает:** Полный массив JSON со всеми блоками.

### 3. Майнинг нового блока
* **Метод:** `POST`
* **Эндпоинт:** `/mine`
* **Тело запроса (JSON):** `{"data": "строка_данных"}`
* **Логика:** Выполняет алгоритм Proof-of-Work (PoW) на стороне сервера.

## 📦 Структура блока (схема данных)

Каждый блок в цепи представляет собой JSON-объект следующей структуры:

```json
{
  "index": 0,
  "previous_hash": "строка_64_символа",
  "timestamp": 1718712345.12,
  "data": "Полезная нагрузка (транзакции/текст)",
  "nonce": 4215,
  "hash": "строка_с_нулями_в_начале"
}
```

## 🤝 Руководство по контрибьютингу (Contribution Guide)

Мы приветствуем улучшение архитектуры ядра! Перед отправкой Pull Request ознакомьтесь с правилами:

1. **Кросс-платформенность:** Запрещено использовать системно-зависимые утилиты (например, `taskkill` или `bash`-скрипты). Используйте только встроенные модули Python (`os`, `sys`, `subprocess`).
2. **Оформление коммитов:** Пожалуйста, используйте понятные описания изменений (желательно по стандарту Conventional Commits, например: `feat: ...`, `fix: ...`, `refactor: ...`).
3. **Чистота кода:** Логика блокчейна и логика интерфейса должны оставаться разделенными между `server.py` и `main.py`.
